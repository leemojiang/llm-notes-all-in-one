
## 样本还是随机变量? 期望和分布的严格Notation

期望角标和括号里的 \\(X\\) 应该大写还是小写？这个问题看似很简单，但实际并没有那么显然。ML 里的符号有很多工程化简写，再加上不同学科的符号习惯并不完全一致，导致同一个符号在不同语境下可能承担不同角色。

符号是定义的体现。所以这一节先从严格的数学视角出发，区分随机变量、取值、样本和分布；再回到 ML 论文中常见的简写，说明这些简写到底省略了什么。

### 1. 分布符号的严格写法

在严格概率论中，“服从某个分布”的对象应该是随机变量，而不是一个具体取值。通常我们用大写字母表示随机变量，用小写字母表示随机变量可能取到的值。

例如，设随机变量 \\(X\\) 的取值空间为 \\(\mathcal{X}\\)，它的概率密度函数或概率质量函数为 \\(p(x)\\)。严格地说，我们可以写：

\\[
X \sim p
\\]

或者在连续分布语境下，为了强调密度函数在点 \\(x\\) 处的值，也常写成：

\\[
X \sim p(x)
\\]


这里各个符号的含义是：

- \\(X\\)：随机变量，也就是从概率空间到取值空间 \\(\mathcal{X}\\) 的映射。
- \\(x\\)：随机变量 \\(X\\) 的一个可能取值，是取值空间里的点。
- \\(p(x)\\)：随机变量 \\(X\\) 的密度函数或质量函数在 \\(x\\) 处的值。
- \\(x_i\\)：一个已经观测到的样本，通常可以理解为某个随机变量 \\(X_i\\) 在一次实验结果 \\(\omega_i\\) 上的取值，即 \\(x_i = X_i(\omega_i)\\)。

所以，从严格意义上说，下面这种写法并不准确：

\\[
x \sim p(x)
\\]

因为 \\(x\\) 是一个取值，不是随机变量。一个点本身不会“服从分布”，只有随机变量才谈得上“服从分布”。

不过在 ML 论文和代码伪代码中，\\(x \sim p(x)\\) 非常常见。它通常不是严格的概率论表述，而是一种工程化简写，真实含义是：

> 从分布 \\(p\\) 中采样一个值，并把这个值记作 \\(x\\)。

因此，在直觉解释或算法伪代码中，写 \\(x \sim p(x)\\) 通常可以接受；但在严谨推导中，最好先区分随机变量和取值，例如写成 \\(X \sim p(x)\\)，再用 \\(x\\) 表示积分变量或具体取值。

### 2. 期望符号的严格写法

期望的括号里放的是随机变量，或者由随机变量构造出来的新随机变量。例如：

\\[
\mathbb{E}[X]
\\]

表示随机变量 \\(X\\) 的期望。如果我们关心的是 \\(X\\) 经过函数 \\(f\\) 之后的结果，则写成：

\\[
\mathbb{E}[f(X)]
\\]

这里 \\(f(X)\\) 仍然是一个随机变量，因为 \\(X\\) 是随机的，\\(f(X)\\) 的值也会随实验结果变化。

如果 \\(X\\) 有密度函数 \\(p(x)\\)，那么期望可以展开成积分形式：

\\[
\mathbb{E}[f(X)] = \int_{\mathcal{X}} f(x)p(x)dx
\\]

这里需要特别注意 \\(X\\) 和 \\(x\\) 的区别：

- \\(X\\)：随机变量。
- \\(x\\)：积分变量，也叫 dummy variable，只是积分表达式里的占位符。
- \\(f(X)\\)：随机变量 \\(X\\) 经过函数 \\(f\\) 得到的新随机变量。
- \\(f(x)\\)：函数 \\(f\\) 在具体取值 \\(x\\) 上的函数值。

所以，\\(\mathbb{E}[f(X)]\\) 和 \\(\int f(x)p(x)dx\\) 是同一个对象的两种表达方式：前者强调随机变量，后者强调对取值空间做积分。

在 ML 中，我们常把上式简写成：

\\[
\mathbb{E}_{x \sim p(x)}[f(x)]
\\]

这是一种非常常见的写法，但它混合了两层含义：

1. 角标里的 \\(x \sim p(x)\\) 像是在说“从分布中采样”。
2. 括号里的 \\(f(x)\\) 又把 \\(x\\) 当成积分变量或样本值。

如果写成积分，它真正对应的是：

\\[
\mathbb{E}_{x \sim p(x)}[f(x)]=\int f(x)p(x)dx
\\]

因此，使用带角标的期望时，一个比较稳妥的规则是：角标里如果写了 \\(x\\)，括号里也使用同一个小写 \\(x\\)，把它理解为积分变量。例如：

\\[
\mathbb{E}_{x \sim p(x)}[f(x)]
\\]

而不要写成：

\\[
\mathbb{E}_{x \sim p(x)}[f(X)]
\\]

后一种写法容易把随机变量 \\(X\\) 和积分变量 \\(x\\) 混在一起。

如果希望既严谨又不啰嗦，可以先写清楚随机变量的分布：

\\[
X \sim p(x)
\\]

然后写：

\\[
\mathbb{E}[f(X)] = \int_{\mathcal{X}} f(x)p(x)dx
\\]

在后续推导中，再根据上下文简写为：

\\[
\mathbb{E}_{x \sim p(x)}[f(x)]
\\]

这样读者既能看到严格对象 \\(X\\)，也能看懂 ML 论文中常见的简写形式。

### 3. 期望角标的本质：对测度积分

这时我们发现一个问题：为什么期望的角标里会出现 \\(x \sim p(x)\\) 这种看起来不太严谨的写法？一个 dummy variable 来自一个分布？这似乎和前面“只有随机变量才能服从分布”的说法矛盾了。

这个问题会把我们引到期望更本质的定义：**期望不是对某个符号本身取平均，而是对某个概率测度做积分**。因此，期望角标 \\(\mathbb{E}_{x\sim p(x)}\\) 本质上是“积分变量 + 积分测度”的简写，不是严格的“\\(x\\) 服从分布”。

### 概率是什么? 测度论视角

在测度论中，一个概率空间写作：

\\[
(\Omega, \mathcal{F}, \mathbb{P})
\\]

其中：

- \\(\Omega\\)：样本空间，表示所有可能的实验结果。
- \\(\mathcal{F}\\)：事件集合。是样本空间上的一个 \\(\sigma\\)-代数，表示我们关心的事件集合。(可测集合)
- \\(\mathbb{P}\\)：定义在事件上的概率。给每个事件分配一个[0,1]之间的数.

可以说概率本质是一个测度,他就是一个给事件赋值的函数,只要满足以下三个条件：
- 非负性：\\(\mathbb{P}(A) \geq 0\\) 对所有事件 \\(A \in \mathcal{F}\\)。
- 规范化：\\(\mathbb{P}(\Omega) = 1\\)。
- 

随机变量 \\(X\\) 并不是一个变量,本质是一个**函数**: 是一个从样本空间到取值空间的映射：

\\[
X: \Omega \to \mathcal{X}
\\]

也就是说，\\(X\\) 会把每个实验结果 \\(\omega \in \Omega\\) 映射成一个取值 \\(x \in \mathcal{X}\\)。




我们常说的概率其实是一种测度,它是这么定义的:

我们可以用 \\( x \in A \\) 定义的集合来表示一个事件,


如果我们关心 \\(X\\) 的取值 \\( x \\) 落在某个集合 \\(A\\) 里的概率，就需要先把 \\(A\\) 拉回到原来的样本空间：

\\[
X^{-1}(A) = \{\omega \in \Omega: X(\omega) \in A\}
\\]

这里 \\(A\\) 是取值空间 \\(\mathcal{X}\\) 中的可测集合，而 \\(X^{-1}(A)\\) 是样本空间 \\(\Omega\\) 中的事件。于是，随机变量 \\(X\\) 会在 \\(\mathcal{X}\\) 上诱导出一个分布测度 \\(\mu\\)：

\\[
\mu(A) = \mathbb{P}(X^{-1}(A)) = \mathbb{P}(\{\omega \in \Omega: X(\omega) \in A\})
\\]

这就是“\\(X\\) 的分布”的测度论表达。平时我们说 \\(X\\) 服从某个分布，本质上就是说 \\(X\\) 在取值空间 \\(\mathcal{X}\\) 上诱导出了某个概率测度 \\(\mu\\)。


### 测度论视角下的期望
从原始概率空间看，\\(f(X)\\) 的期望定义为：

\\[
\mathbb{E}[f(X)] = \int_{\Omega} f(X(\omega))\,d\mathbb{P}(\omega)
\\]

而由于 \\(X\\) 已经把 \\(\mathbb{P}\\) 推到了取值空间 \\(\mathcal{X}\\) 上，这个积分也可以写成对分布测度 \\(\mu\\) 的积分：

\\[
\mathbb{E}[f(X)] = \int_{\mathcal{X}} f(x)\,d\mu(x)
\\]

这才是期望更本质的表达。这里没有 \\(\sim\\)，也没有“dummy variable 服从分布”的说法，只有函数 \\(f\\) 和测度 \\(\mu\\)。

如果 \\(\mu\\) 可以写成密度函数 \\(p(x)\\) 的形式，也就是：

\\[
d\mu(x) = p(x)dx
\\]

那么期望就变成我们更熟悉的积分形式：

\\[
\mathbb{E}[f(X)] = \int_{\mathcal{X}} f(x)p(x)dx
\\]

现在再回头看 ML 中常见的写法：

\\[
\mathbb{E}_{x \sim p(x)}[f(x)]
\\]

它严格展开后对应的是：

\\[
\int_{\mathcal{X}} f(x)p(x)dx
\\]

或者更本质地说，对应的是：

\\[
\int_{\mathcal{X}} f(x)d\mu(x)
\\]

所以，\\(\mathbb{E}_{x \sim p(x)}[f(x)]\\) 中的角标并不是严格地说“\\(x\\) 服从分布 \\(p\\)”。它更像是把下面两件事压缩到一个角标里：

- 积分变量叫 \\(x\\)。
- 积分所使用的分布或测度由 \\(p\\) 给出。

因此，\\(x \sim p(x)\\) 出现在期望角标里时，不必把它理解为严格的“取值 \\(x\\) 服从分布”。它只是 ML 语境中约定俗成的 shorthand，用来提醒读者：这里要对分布 \\(p\\) 下的取值进行积分或平均。

如果想写得既严谨又不啰嗦，可以在第一次出现时写清楚：

\\[
X \sim p(x), \quad \mathbb{E}[f(X)] = \int_{\mathcal{X}} f(x)p(x)dx
\\]

之后再使用：

\\[
\mathbb{E}_{x \sim p(x)}[f(x)]
\\]

并把它理解为“对密度 \\(p(x)\\) 积分，\\(x\\) 是积分变量”。

### 小结

严格写法中，随机变量和取值需要区分：

\\[
X \sim p(x),\quad x \in \mathcal{X}
\\]

期望的抽象写法强调随机变量：

\\[
\mathbb{E}[f(X)]
\\]

积分写法强调取值空间上的 dummy variable：

\\[
\mathbb{E}[f(X)] = \int_{\mathcal{X}} f(x)p(x)dx
\\]

ML 中常见的简写：

\\[
\mathbb{E}_{x \sim p(x)}[f(x)]
\\]

本质上是对积分形式的压缩。它好用、直观，但并不是最严格的概率论符号。做理论推导时，最好先用 \\(X\\) 表示随机变量，用 \\(x\\) 表示取值或者积分变量,用 \\(x_i\\) 表示样本，再根据上下文决定是否采用 ML 风格的简写。
