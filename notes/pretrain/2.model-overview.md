# 模型总览: 一个预训练模型由哪些部分组成?

在真正进入某个模块之前，最好先回答一个更整体的问题：

1. 如果从代码实现的角度看，一个预训练模型通常由哪些部分组成？
2. 哪些部分是“模型定义”本身，哪些部分是“训练流程”中的配置？
3. 为什么很多看起来像局部实现细节的选择，实际上会影响模型能力、训练稳定性和推理效率？

这一节先不给出所有模块的完整推导，而是先建立一张整体地图。后面真正进入 Attention、RoPE、训练目标时，再分别展开。

## Q1: 一个预训练模型，最少需要哪些部分?

如果先站在一个很粗略的层面去看，一个可以运行的预训练模型，至少会涉及这些部分：

1. Tokenizer
2. Embedding
3. 多层 Transformer Block
4. 最终的输出层
5. 与训练相关的损失函数、优化器和数据输入

其中，这一节先重点关注前四项，也就是模型定义本身。

## Q2: 模型定义里最常见的几个问题是什么?

结合 MiniMind 的代码和我自己的整理，我现在更倾向于把模型定义阶段的问题先列成下面这样：

### 整体设置

- 激活函数用什么？
- 层间的 Norm 用什么？
- Dropout 放在哪些位置？比例怎么设置？

### Embedding 层

- token id 是如何变成 embedding 的？
- embedding 矩阵为什么可以理解成“词表中的每个 token 对应一行向量”？

### FFN / MLP 层

- FFN 在 Transformer block 里扮演什么角色？
- 中间层维度通常为什么要比 hidden size 更大？

### Attention 层

- Position Encoding 怎么实现？
- 为什么现在很多模型使用 RoPE，而不是最早那种直接加到输入上的位置编码？
- 为什么 RoPE 作用在 Q/K 上，而不是直接作用在输入 embedding 上？
- 多头注意力里的 head_dim 一般怎么选？
- 为什么 Q heads 和 KV heads 可以不一样？

这些问题里，Attention 相关的内容最多，也最值得单独拆出来讲，所以放到下一节重点展开。

## Q3: Embedding 层在做什么?

模型并不能直接处理原始字符串，它真正接收的是 tokenizer 处理后的 token id。

假设输入序列写成：

$$
t_1, t_2, \dots, t_n
$$

其中：

- $t_i$ 表示第 $i$ 个 token 对应的 id
- $n$ 表示序列长度

Embedding 层的作用，就是把这些离散的整数 id 映射成连续向量。  
如果词表大小是 $V$，隐藏维度是 $d$，那么 embedding 矩阵通常可以写成：

$$
E \in \mathbb{R}^{V \times d}
$$

其中：

- $V$ 表示词表大小
- $d$ 表示 embedding 维度

这样一来，每个 token id 本质上就是在 embedding 矩阵里“取一行”。  
所以在直觉上完全可以把 embedding 理解成：

- 词表里的每个 token
- 都对应一个可学习的向量表示

这也是为什么 embedding 层虽然看起来简单，但它实际上已经是模型参数的一部分。

## Q4: FFN / MLP 层在做什么?

Attention 负责让不同 token 之间发生信息交互，但如果只有 attention，模型表达能力还是不够。

Transformer block 里通常还会有一个前馈网络，也就是 FFN 或 MLP。它的常见形式可以写成：

$$
\mathrm{FFN}(x) = W_2 \,\phi(W_1 x + b_1) + b_2
$$

其中：

- $x$ 表示输入向量
- $W_1, W_2$ 表示线性变换矩阵
- $b_1, b_2$ 表示偏置项
- $\phi$ 表示激活函数，比如 GELU、SiLU

这个模块可以粗略理解成：  
在每个 token 自己的特征维度上，再做一次更强的非线性变换。

这也是为什么在很多实现里，FFN 的中间维度会明显大于 hidden size。因为它需要在更高维空间里完成特征变换，再投回原维度。

## Q5: 激活函数、Norm、Dropout 为什么值得单独问?

一开始很容易觉得这些只是工程参数，但实际不是。

### 激活函数

激活函数决定了 FFN 里的非线性形式。不同激活函数会影响：

- 表达能力
- 梯度性质
- 训练稳定性

在今天的 LLM 里，常见的选择往往不是最早期的 ReLU，而更常见是 GELU、SiLU 这类更平滑的函数。

### Norm

Norm 的核心作用，是帮助训练更稳定。  
从实际实现看，LayerNorm、RMSNorm 都很常见，而不同模型会在：

- 用哪一种 Norm
- 放在 block 的什么位置

上做不同取舍。

### Dropout

Dropout 则更偏训练阶段的正则化。  
它最值得问的不是“有没有这个 API”，而是：

- 放在哪些位置？
- 比例设多少？
- 在小模型和大模型里，这个设置是不是一致？

这些问题表面上是实现细节，实际上经常会影响训练行为。

## Q6: 为什么这一节还不直接推 Attention 公式?

因为 Attention 既是模型结构的一部分，又几乎是整个 LLM 主干里最核心的模块。  
如果把它直接混在“模型模块概览”里讲，很容易两边都讲不透。

所以这一节的目标只是先明确：

- 模型不是只有 Attention
- 但 Attention 是最核心的部分
- 它和 Embedding、FFN、Norm、位置编码一起，构成了今天 LLM 的主干结构
