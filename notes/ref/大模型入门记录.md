# Roadmap For LLM beginners

## 写给谁?
这篇文章作为最近一个月的学习总结,适用对象为想要入门LLM的新人.入门一个领域有时候是最困难的,直接扑在具体的细节上很可能会迅速迷失其中,所以论文或者别人的学习笔记可能并不是很好的学习资料.我对此的思考是与其解答具体的知识内容,一份Roadmap可能是更好的选择.它就像是课程的大纲或者书籍的目录,罗列出有哪些知识点,并给出答案的引用.我会按照一系列渐进问题的形式来组织这篇文章.

## 哪些资料适合入门学习?
我觉得这是一个非常重要的问题,也是这次学习的一大启发,即如何从海量资料里选出适合自己学习的资料,或者说好的入门学习材料应该有哪些性质?
    
1. 系统体系化的知识内容.
    具体来说: 
    知识内容全面(知识点都涉及到,定义清晰,概念自洽,不会出一堆看不懂的名词).
    最好再把点之间逻辑/发展脉络/联系梳理清楚.

2. 有具体的实践内容,可以实际上手操作.
    可以写一些代码,或者做一些事情. 
    实践操作对学习非常必须.

## LLM学习的第一步从哪里开始?
对于学习的第一步,我推荐做一些能在1-2两天内看到反馈的事情,而不是先学习一个月的课程和理论知识,那样等到实践的的时候大概率忘得差不多了.本地部署一个LLM比如最近很火的DeepSeek是一个非常好的选择,这个过程能快速对大模型应用建立直观的印象.

- 一个大模型应用包含哪些部分?  
    前端交互(Open WebUI/ChatBox/CherryStudio)  
    后端推理框架(Ollama)  
    - (进阶问题) 后端推理框架如何加速? 
    - (进阶问题) 对于参数量超过单一显存的显卡,如何调度多块显卡?
    - (进阶问题) 如何应对并发用户? 

- 前后端之间使用的API是什么格式的?  

    完全可以使用其他在线API,并且大部分API都兼容OpenAI API.前后端分离可以很容易通过API与不同的模型沟通.比如可以往Cursor里面接入DeepSeek.

- 部署模型的时候,模型的输入输出是什么样子的?  
    - Ollama的model template是怎么定义的?
    - ChatML(Chat Markup Language)是什么?  
        - (进阶问题) ChatML做了什么操作来保证安全性?
    
        到这一步会发现不同的输入对模型的影响非常大,比如我在本地部署的时候出现过model template配置错误,模型直接胡说八道,算是初步领略了Prompt Engineering 的魅力.

- 怎么让模型使用更精准的信息/本地信息?  
    这个问题自然而然引入了RAG,不得不说RAG的思路真的简单粗暴.其实到这一步已经可以尝试去搭建一些好玩的LLM应用了.

    - (进阶问题) RAG数据库如何加速访问速度以应对大规模数据?
    - (进阶问题) RAG的信息检索部分有没有优化空间?   
        比如更好的embedding,Graph RAG替代简单的向量相似度.

## 基础知识到前GPT时代(Pre 2020)?
开始进入一些更深入的内容,这部分包括一些基础的NLP知识以Transformer(2017)/Attention(2014)基础.不知不觉都是10年前的内容了,这些几乎都是NLP领域上个时代的研究范式了,但是对于NLP领域之外的人这些东西还有点陌生,这个时代的结束大概可以概括为结束在2020年之前,那时BERT(2018)还是大热点,GPT2(2019)才崭露头角,但是很多基础和经典的算法依然沿用至今.

对于这类基础问题,包括经典的入门问题:Transformer是什么?我建议与NLP课程一起学习,虽说这些东西都很基本,但是没有系统学习依然难以理解其中的发展流程,以及背后提出的思想.并且一些NLP的基础操作和标准流程依然沿用至今.所以我非常推荐有关注性的学习一下NLP课程.我当时直接看帖子学习Transformer也是百思不得其解,但通过NLP课程系统的学习之后 (word2vec->word embedding-> seq2seq -> attention-> transform) 明白了NLP每一阶段或步骤都是为了解决什么问题,之后的发展又是如何对前者进行改进,其中的逻辑就会非常自然.当然像Parsing之类的与LLM关系不大的内容,也可以跳过.

或者也可以直接读这篇神文,这篇文章不但做完美的做到了理论结合实践,同时也涉及了LLM需要用到的必备NLP知识(tokenizer/embedding),做到一篇入门,属于必读的佳作.
[GPT in Numpy](https://jaykmody.com/blog/gpt-from-scratch/)

- 什么是Language Model?  
    Large Language Model 依然是 LM,LM是一个概率模型.

- LM的输入和输出是什么?/NLP的经典处理流程是什么?  
    输入理论上是embedding,实际是tokenID.输出就开始牵扯复杂的细节,这里列举几个术语:logits(模型的原始输出),language modeling head(原始输出后面接的部分,比如使用投影矩阵把logits转化为LM的概率.)
    - (细节) NLP中有哪些传统任务,这些任务是怎么做的?
        [HuggingFace](https://huggingface.co/learn/llm-course/chapter1/3?fw=pt)

    - (细节) Tokenizer是什么?怎么工作的?  
        输入是完整的句子,通过词表(vocab)把单词切开,可能到subword级别,比如BPE算法.最后返回的是vocab里的index.

        这部分也可以补充参考[HuggingFace NLP Course](https://huggingface.co/learn/llm-course/chapter2/2?fw=pt)

    - (细节) TokenID是如何变成embedding的?  
        GPT2中embedding是训练出来的,作为模型参数的一部分.  
        - (进阶问题) 传统Word2Embedding算法是怎么做的?
    
    - (细节) 对于Transformer Position Embedding为什么需要?又有哪些问题?
        Position embedding也是训练的,没有这个Transformer无法识别句子顺序,但是固定的position embedding矩阵限制了模型的最大输入范围.
        - (进阶问题) Position Embedding还有哪些实现方式?这些不同实现方式之间的区别是什么? 如何摆脱输入长度限制?

- Attention/Transfomer的实现细节是什么样子的?  
    这部分非常繁琐,细节很多,经常觉得懂了但其实没办法手写公式,更别提写代码.需要经常用输出来检测.整个推导流程时刻关注输入输出的维度是个比较好的方法,[这里有非常好的可视化.](https://zhuanlan.zhihu.com/p/4586600974)

    - (细节) Attention(2014)提出的思想?  
        从期望的角度理解,非常合理.
    - Multi-head self attention 的self,以及multi-head是怎么实现的?
    - Attention是如何实现casual时序的?Masking是如何实现的?
    - Transformer中引入了哪些细节来技巧来辅助训练?  
        LayerNorm/softmax归一化等
    - Attention的计算复杂度是多少? 如何加速?
    - (进阶问题) 有哪些Attention改进,这些改进的目的是什么?他们的发展联系又是什么?

- GPT/Transformer是怎么实现以及训练的?
    - Auto Regressive训练范式是怎么做的?  
        自监督做到scaling up
    - (进阶问题) 其他后续模型上训练目标和方法的改良?
    - (细节) 早期Pretrain-Finetune范式是什么样子的?
